Brief

A database is an organized collection of structured data that is typically stored electronically in a computer system. Data within the most common form of databases today is typically done in rows and columns with a series of tables to make processing and data querying efficient. This also allows for data can then be easily accessed, managed, modified, updated, controlled, and organized. Most databases use structured query language (SQL) for writing and querying data. In this internal we have created a database about books we have included information such as the authors name, book genre etc. This database has been created with the intention of organising a book club through the data base we can record different books and their associated information. We will also be able to record who in the book club has read what boook whitch will helpful so that nobody has to read the same book twice. If someone was to read a book but for example forgot the author and its title then as all books in the book club are stored on the data base they would be able to find the book by using querys such as selecting only books with a specific genre.

Relevant Implications - Description

Functionality

For the relevant implication of functionality to be met there must be no errors and no incorrectly entered data in the database. The database should work as expected meaning the user can get any information in any combination.

Future Proofing

The future proofing implication can be met if databases are designed in a way that allows for them to be updated and have features added to it. The database should also still function as expected if data is added or removed.

Usability

The implication of usability is a measure of how well a user can use a product to achieve a defined goal effectively, efficiently and satisfactorily. To ensure usability in a database information is displayed in a comprehensible way.
